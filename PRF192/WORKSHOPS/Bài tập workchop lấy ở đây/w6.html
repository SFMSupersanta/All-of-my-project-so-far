Module E - Libraries

Simple Games

Workshop 6


In this workshop, you are to design a simple game that includes robust input validation. 


Learning Outcome

Upon successful completion of this workshop, you will have demonstrated the abilities

  • to validate input robustly, and
  • to use library functions.

Dice Throws

Design and code a program that throws two dice until the top faces of the two dice total to a specified number.  The values on each die range from one to six inclusive.  Your program prompts for and accepts the total number sought, includes input validation and uses both your own and standard library functions. 

Design your program according to structured design principles and write your own functions so that you can readily use them in other applications. 

Preface your function headers with comprehensive descriptions of function purpose, function parameters and function return value.

The output from your program looks something like:  

 Dice Thrower
 ============
 Total sought : 11
 Result of throw  1 : 1 + 3
 Result of throw  2 : 4 + 4
 Result of throw  3 : 6 + 2
 Result of throw  4 : 5 + 6
 You got your total in 4 throws!


Ball Lottery

Design and code a program that keeps picking two balls at a time from a basket of ten until the values on the pair of picked balls totals a specified number.  The value on each balls lies between one and ten inclusive.  Your program prompts for and accepts the total number sought, includes input validation and uses your own and standard library functions. 

Design your program according to structured design principles and write the functions so that you can readily use them in other applications. 

Preface your function headers with comprehensive descriptions of function purpose, function parameters and function return value.

The output from your program looks something like:  

 Ball Lottery
 ============
 Total sought : 11
 Result of picks 1 and 2 : 1 + 3
 Result of picks 3 and 4 : 4 + 5
 Result of picks 5 and 6 : 6 + 3
 Result of picks 7 and 8 : 5 + 6
 You got your total in 8 picks!


Card Draw

Design and code a program that keeps drawing a card from an original deck of 52 cards until the card drawn is the card requested.  Your program prompts for and accepts the suit and rank of the card sought, includes input validation and uses your own and standard library functions.  You may assume that the user returns the card drawn to the deck before drawing another card.

Design your program according to structured design principles and write the functions so that you can readily use them in other applications. 

Preface your function headers with comprehensive descriptions of function purpose, function parameters and function return value.

The output from your program looks something like:  

 Card Draw
 =========
 Suit : H
 Rank : 4
 Result of draw  1 : 2C
 Result of draw  2 : 3S
 Result of draw  3 : KD
 Result of draw  4 : 4H
 You got your result in 4 draws!
