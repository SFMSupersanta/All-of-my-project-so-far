Assignment 3
Version 1.0

Payroll Deductions
with Functions and Robust Validation



As your third assignment this semester, you are to subdivide your Payroll Deductions code into functions and to upgrade the validation logic to accomodate any input whatsoever. 


Learning Outcomes

Upon successful completion of this assignment, you will have demonstrated the ability to use functions as well as sequence, selection and iteration structures.


Function Design

Upgrade your program so that another programmer can modify it with minimal intrusion.  Divide your code into functions to achieve the highest possible cohesion and the lowest possible coupling.  For guidelines on designing functions see the Modularity reading. 

Your program asks the user whether or not to continue with another run, and if the user answers yes, your program continues without terminating execution.

Your program includes no less than 8 functions.  Marks will be allocated for quality designs. 


Robust Validation

Your program accepts and validates any input whatsoever from the user.  DO NOT assume that the user will only enter numbers, whitespace or possibly a decimal point, but no other characters.  Your program generates an appropriate response for any erroneous input.  Examples of unacceptable or unreasonable input include:

  • input is negative in value
  • RRSP contribution exceeds the income
  • Union dues exceed the income less the RRSP contribution
  • a decimal point appears in integer input
  • more than one decimal point appears in floating-point input
  • characters are embedded in numeric input

 Have a Nice Day!

Place the function prototypes and the constant definitions in a header file with the same name as your source code file and use a .h suffix on the file name rather than a .c suffix.  For example, if the name of your source file is jdoe3.c, name your header file jdoe3.h
