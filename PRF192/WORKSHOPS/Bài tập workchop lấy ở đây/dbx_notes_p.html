dbx Debugging Tool on AIX



 
Introduction

Most compilers/operating systems come with a debugging tool.  The IBM version of the UNIX operating system (AIX) on phobos comes with the dbx command.

dbx provides an environment to debug and run C programs.


Compile and Run

To use dbx, first compile your program with the -g option 


 cc -g myProgram.c

This will produce an executable called a.out that can be debugged.  To debug your executable, enter 

 dbx a.out

This will display an introductory message followed by the dbx prompt 

 Type 'help' for help.
 reading symbolic information ...
 (dbx)

Type your dbx commands at this prompt.


Commands

The dbx commands include:

  • list - lists the next 10 lines of source code starting from the current line.  Each call advances the current line by 10
  • list n - where n is a line number - lists line n of the source code.  This call advances the current line by 1
  • list m, n - where m and n are line numbers - lists lines m through n inclusive of the source code.  This call advances the current line to n+1
  • stop at n - where n is a line number - sets a break point at line number n
  • clear n - where n is a line number - clears any break point or trace at line number n
  • trace varname - where varname is a variable name - displays changes in the value of varname
  • delete all - clears all traces and breakpoints
  • status - displays all trace and breakpoint information
  • run - starts the execution of your program from line 1
  • print varname - where varname is a variable name - displays the value of the variable varname
  • cont - continues execution until either your program ends or a breakpoint is encountered
  • step - executes one line of your program
  • help - displays the full set of commands available
  • quit - quits
dbx, like C, is case sensitive.

When you first start dbx, your program pauses.  This is your first opportunity to set your breakpoints.  Enter the run command only once you are ready to start execution.


Crashes

If your program fails and produces a core dump, dbx can help you find out where your program crashed.  Enter 


 dbx a.out core

The following commands help to investigate crashes
  • where - displays the function and line number where your program was executing at the time of the crash
  • up - moves up one function in the stack (towards main)
  • down - moves down one function in the stack (away from main)

Help

For online help with a particular command while debugging, enter 


 help command

at the dbx prompt, where command is the command in question.

For online help with the dbx command while not debugging, enter 


 man dbx

at the phobos prompt.
